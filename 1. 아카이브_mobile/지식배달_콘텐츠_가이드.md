# 지식 배달 콘텐츠 생성 가이드 (매일 생성용)

> 직장인/수험생 배달 콘텐츠를 **어디에 · 어떤 형식으로 · 어떤 id로** 추가하는지의 단일 기준 문서.
> 매일 조금씩 생성할 때 아래 §4의 "복붙 프롬프트"를 다른 대화에 붙여넣고 주제만 바꾼다.
> **기준일 2026-06-24** — 아래 "다음 id"는 이 날짜 기준. 새 작업 전 `npm run scan-ids`로 항상 재확인.

---

## 1. 저장 위치 (3개 폴더)

| 모드 | 폴더 | 적재 키 |
|---|---|---|
| **직장인** | `public/data/knowledge_db/*.json` | english_theme_packs · english_expressions · chinese_expressions · history_facts |
| **직장인 (신규)** | `public/data/work_db/*.json` | classic_quotes(고전) · idiom_cards(고사성어) · daily_insights(인사이트) |
| **수험생** | `public/data/exam_db/*.json` | exam_vocab_themes · exam_history_items |

- 서버가 폴더 전체 `.json`을 읽어 메모리/SQLite에 적재(중복 id는 자동 스킵). 배달은 AI 호출 없이 회전 서빙.
- `archive.db` · `dailyFeeds.json` 은 **런타임 생성물 — 직접 편집 금지.** 콘텐츠는 항상 위 시드 폴더에만.
- **모드 격리:** 수험생 배달은 `exam_db`만 읽는다. `knowledge_db`·`work_db`(직장인)는 수험생에 **절대 안 뜸**. 직장인 콘텐츠를 `exam_db`에 넣지 말 것(그 반대도).

> ⚠️ 폐기됨(더 넣지 말 것): 옛 `idioms_and_quotes`(IQ_) / `quote_daily`(명언) — 고사성어는 이제 `work_db/idiom_cards`로, 명언은 고전(LIBER)·인사이트로 대체됨.

---

## 2. 절대 규칙
1. **최상위 키 이름 정확히** (오타 시 미적재).
2. **id 전역 고유** — 이미 쓴 번호 재사용 금지(`INSERT OR IGNORE`라 조용히 누락). 항상 "다음 id"부터.
3. **파일 이름 자유** (폴더 전체를 읽음). 관례: knowledge_db=`batch_NN_주제.json`, work_db=`<타입>_NN.json`, exam_db=`vocab_NN_csat.json`/`history_NN_kr.json`.
4. work_db 항목은 `targetMode: "work"`, `used: false` 고정.
5. JSON에 `meta`(작성일·주제 메모)는 넣어도 무시됨 — 자유롭게.

## 4. 복붙용 생성 프롬프트 (필요한 블록만 복사 → 새 대화에 붙여넣기 → 주제/다음 id만 교체)

### (A) 직장인 — 고전 LIBER  → `work_db/classic_quotes.json`
```
SJ 아카이브 직장인 배달 "고전 LIBER" 콘텐츠를 생성한다. JSON 한 덩어리만 출력, 설명·마크다운 금지.
- 위치: public/data/work_db/ 의 .json, 최상위 키 "classic_quotes"
- id: CLQ_<다음 번호>부터 전역 고유(현재 다음 CLQ_007). 기존과 겹치지 말 것.
- 책 1권에서 직장인에게 울림 주는 구절 6개 선정. 1~4문장, 저장·공유하고 싶을 만큼 인상적.
- 책: <여기에. 예: "논어 / 공자">
스키마:
{ "meta":{"book":"...","created":"YYYY-MM-DD"},
  "classic_quotes":[ {"id":"CLQ_###","book":"책","author":"저자","era":"사조(영문)","quote":"구절 한국어","source":"출처(예: 제4권 7절)","theme":"핵심 주제 한 단어","context":"직장인 관점 해석 2~3문장","tags":["철학","리더십"],"targetMode":"work","used":false} ] }
```

### (B) 직장인 — 고사성어  → `work_db/idiom_cards.json`
```
SJ 아카이브 직장인 배달 "고사성어" 콘텐츠를 생성한다. JSON 한 덩어리만 출력, 설명·마크다운 금지.
- 위치: public/data/work_db/ 의 .json, 최상위 키 "idiom_cards"
- id: IDC_<다음 번호>부터 전역 고유(현재 다음 IDC_003).
- 직장·비즈니스 관점으로 풀이. 성어 5개.
- 주제 영역: <여기에. 예: "리더십·전략">
스키마:
{ "meta":{"topic":"...","created":"YYYY-MM-DD"},
  "idiom_cards":[ {"id":"IDC_###","idiom":"새옹지마","hanja":"塞翁之馬","meaning":"뜻 1~2문장","origin":"유래·출전 1~2문장","example":"이 성어를 쓴 실제 문장 1개","modernUse":"직장에서 쓰는 구체적 상황 2~3문장","contextType":"work","tags":["인내","전략"],"targetMode":"work","used":false} ] }
```

### (C) 직장인 — 오늘의 인사이트  → `work_db/daily_insights.json`
```
SJ 아카이브 직장인 배달 "오늘의 인사이트" 콘텐츠를 생성한다. JSON 한 덩어리만 출력, 설명·마크다운 금지.
- 위치: public/data/work_db/ 의 .json, 최상위 키 "daily_insights"
- id: INS_<다음 번호>부터 전역 고유(현재 다음 INS_003).
- 카테고리(요일)별로 작성. 개념 5개. 직장인이 3분 안에 이해하고 써먹게.
- 카테고리 메타(고정): 심리학(psychology,🧠,#7c3aed,월=1) / 행동경제학(behavioral_economics,💹,#059669,화=2) / 철학 한 줌(philosophy,🏛️,#92400e,수=3) / 조직·사회학(sociology_org,🏢,#0891b2,목=4) / (금·토·일은 자유 확장)
- 이번 카테고리·주제들: <여기에. 예: "심리학 / 손실 회피, 밴드왜건 효과 …">
스키마:
{ "meta":{"category":"...","created":"YYYY-MM-DD"},
  "daily_insights":[ {"id":"INS_###","subCategory":"psychology","label":"심리학","icon":"🧠","color":"#7c3aed","dayOfWeek":1,"topic":"확증 편향","headline":"한 줄 핵심(호기심 자극)","body":"개념 설명 3~4문장","realLife":"직장 사례 2~3문장","question":"오늘 스스로 던질 질문 1개","tags":["인지편향","의사결정"],"targetMode":"work","used":false} ] }
```

### (D) 직장인 — 영어 테마팩  → `knowledge_db/batch_NN_*.json`
```
SJ 아카이브 직장인 배달 "영어 테마팩"을 생성한다. JSON 한 덩어리만 출력, 설명 금지.
- 위치: public/data/knowledge_db/ 의 .json, 최상위 키 "english_theme_packs"
- id: PACK_EN_<다음 번호>부터(현재 다음 PACK_EN_028), 표현 id "<팩id>_01"…"_05". 전역 고유.
- 한 팩 = 실전 상황 테마 1개 + 표현 5개 + 마스터 단락. dialogue 3~4턴(\n 줄바꿈), nuance는 "왜 이 표현"을 2~4문장.
- 주제: <여기에. 예: "재택·하이브리드 근무 커뮤니케이션">
스키마:
{ "meta":{"topic":"...","created":"YYYY-MM-DD"},
  "english_theme_packs":[ {"id":"PACK_EN_###","theme_key":"snake_case","theme_title":"한국어 제목","theme_title_en":"English Title","level":"intermediate","delivery_date":null,
    "expressions":[ {"id":"PACK_EN_###_01","order":1,"expression":"...","meaning":"...","nuance":"...","dialogue":"A: ...\nB: ...","example":"...","practice":"..."} ],
    "master_paragraph":{"text":"5~7문장 영어","translation":"한국어 번역","highlights":["표현1","표현2"]} } ] }
```
> 영어 평면 표현(`english_expressions`, 다음 EN_326)·중국어(`chinese_expressions`, 다음 ZH_078)·역사(`history_facts`, 다음 HI_088)는 기존 batch 파일 스키마를 그대로 따른다.

## 5. 추가 후 체크리스트
```
□ 올바른 폴더인가? (직장인=knowledge_db/work_db, 수험생=exam_db)
□ 최상위 키 정확? (오타 시 미적재)
□ id가 "다음 id"부터 시작하고 기존과 안 겹치는가? (npm run scan-ids 로 확인)
□ JSON 문법 OK? (node -e "require('./파일.json')")
□ work_db 항목에 targetMode:"work", used:false 들어갔는가?
□ 서버 재시작 후 콘솔 로그로 적재 확인:
   [WorkDB] 로드 CLQ:.. IDC:.. INS:..  /  [EnTheme Seed] ..  /  [ExamKnowledge Seed] ..
```

> **다음 id 재확인:** `cd "1. 아카이브_mobile" && npm run scan-ids`
> **자동 일괄 생성(선택):** `node scripts/buildWorkContent.js all`(파일럿) / `... all --full`(전체) — Claude 기본, work_db로 출력.

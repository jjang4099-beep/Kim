# 지식 배달 콘텐츠 추가 가이드

> 직장인/수험생 모드의 "지식 배달" 콘텐츠를 **어디에 · 어떤 형식으로** 추가해야 하는지 정의한다.
> 다른 대화(콘텐츠 생성)에서 이 문서의 "복붙용 프롬프트"를 그대로 사용하면 항상 올바른 위치·형식으로 쌓인다.

---

## 1. 저장 구조 한눈에

| 구분 | 시드 소스 (← 여기에 콘텐츠 추가) | 런타임 적재 (자동) |
|---|---|---|
| **직장인** | `public/data/knowledge_db/*.json` | `archive.db` → `english_themes` · `english_expressions` |
| **수험생** | `public/data/exam_db/*.json` | `archive.db` → `exam_vocab_themes` · `exam_vocab_words` · `exam_history_items` |
| 배달 캐시 | (자동 생성) `public/data/dailyFeeds.json` | — |

- 서버는 시작 시 폴더 안 **모든 `.json`을 읽어 최상위 키별로 합쳐** SQLite에 `INSERT OR IGNORE`로 적재한다.
  (`server.js`의 `_seedEnglishThemes()`, `_seedExamKnowledge()`, `loadKnowledgeDB()`)
- 배달은 AI 호출 없이 이 시드 DB를 **회전(rotation)** 서빙한다. → 콘텐츠를 늘리려면 **시드 JSON에 추가**하면 된다.
- `archive.db` / `dailyFeeds.json` 은 **런타임 생성물** — 직접 편집 금지. 콘텐츠는 항상 시드 JSON으로만 넣는다.

---

## 2. 절대 규칙 (이것만 지키면 안전)

1. **최상위 키 이름이 정확해야 한다.** (아래 표 외 키는 무시됨)
2. **`id`는 전역 고유**해야 한다. 이미 쓴 id를 다시 쓰면 `INSERT OR IGNORE`로 **조용히 누락**된다.
   - 새 콘텐츠는 항상 다음 번호로 (예: `EN_044`, `PACK_EN_007`, `EV_PACK_013`, `EH_049`).
3. **파일 이름은 자유** (`batch_07_*.json`, `vocab_03_csat.json` 등). 폴더 전체를 읽으므로 새 파일/기존 파일 어디든 OK.
   다만 일관성을 위해 기존 네이밍(`batch_NN_주제.json`, `vocab_NN_csat.json`, `history_NN_kr.json`)을 따른다.
4. JSON에 `meta` 객체(작성일·주제 메모 등)는 넣어도 무시되니 자유롭게 사용.

### 직장인 `knowledge_db` 최상위 키
| 키 | 용도 | 배달 노출 |
|---|---|---|
| `english_theme_packs` | **영어 메인 경로** — 테마 1개 = 표현 5개 + 마스터 단락 | 우선 서빙 (지식배달의 풍부한 형식) |
| `english_expressions` | 영어 표현 평면 풀 | 테마팩 부족 시 fallback |
| `chinese_expressions` | 중국어 표현 | 중국어 배달 |
| `idioms_and_quotes` | 고사성어·명언 | 명언/고사성어 배달 |
| `history_facts` | (직장인) 역사 상식 | 역사 배달 |

### 수험생 `exam_db` 최상위 키
| 키 | 용도 |
|---|---|
| `exam_vocab_themes` | 수능 영단어 팩 (팩 1개 = 단어 5개) |
| `exam_history_items` | 한국사 항목 |

## 3. 스키마 (실제 필드명)

### 3-1. 영어 테마팩 — `english_theme_packs` (직장인 메인)
```json
{
  "english_theme_packs": [
    {
      "id": "PACK_EN_007",                       // 전역 고유. PACK_EN_### 규칙
      "theme_key": "negotiation_under_pressure",  // 영문 스네이크
      "theme_title": "협상에서 밀리지 않고 주도권을 잡을 때",
      "theme_title_en": "Holding Your Ground in a Tough Negotiation",
      "level": "intermediate",                    // beginner|intermediate|advanced
      "delivery_date": null,                       // 특정일 지정배달 원하면 "YYYY-MM-DD", 보통 null
      "expressions": [
        {
          "id": "PACK_EN_007_01",                 // 팩id_순번
          "order": 1,
          "expression": "Let's meet in the middle.",
          "meaning": "서로 조금씩 양보합시다.",
          "nuance": "왜 이 표현을 쓰는지, 어떤 뉘앙스·상황인지 2~4문장 스토리텔링.",
          "dialogue": "A: ...\nYou: Let's meet in the middle.\nA: ...",   // \n 줄바꿈, 3~4턴
          "example": "After an hour of back-and-forth, we decided to meet in the middle.",
          "practice": "Could we meet in the middle on the delivery timeline?"
        }
        // ... 표현 5개 권장
      ],
      "master_paragraph": {
        "text": "위 표현들을 자연스럽게 엮은 영어 문단 (5~7문장).",
        "translation": "그 문단의 한국어 번역.",
        "highlights": ["meet in the middle", "..."]   // 문단 속 강조할 표현들
      }
    }
  ]
}
```

### 3-2. 영어 표현 평면 풀 — `english_expressions` (fallback)
```json
{
  "english_expressions": [
    {
      "id": "EN_044",                    // EN_### 규칙
      "expression": "Let's table this for now.",
      "meaning": "이 사안은 일단 보류합시다.",
      "level": "intermediate",
      "theme": "meeting",                // 영문 키 (meeting/negotiation/strategy 등)
      "example": "We're running out of time, so let's table this for now.",
      "dialogue": "A: ...\nB: ..."
    }
  ]
}
```

### 3-3. 수능 영단어 팩 — `exam_vocab_themes` (수험생)
```json
{
  "exam_vocab_themes": [
    {
      "id": "EV_PACK_013",               // EV_PACK_### 규칙
      "theme_title": "빈칸 추론 핵심 형용사",
      "level": "high",
      "tip": "이 팩을 왜·언제 쓰는지 한 줄 학습 팁.",
      "delivery_date": null,
      "words": [
        {
          "id": "EV_073",                // EV_### 규칙 (단어별 전역 고유)
          "order": 1,
          "word": "prevalent",
          "pos": "형용사",
          "meaning": "만연한, 널리 퍼진",
          "example_en": "The use of smartphones has become prevalent among all age groups.",
          "example_ko": "스마트폰 사용이 모든 연령대에 걸쳐 만연해졌다.",
          "csat_ref": "2023학년도 수능 34번 유형",
          "synonyms": "widespread, common, dominant"
        }
        // ... 단어 5개 권장
      ]
    }
  ]
}
```

### 3-4. 한국사 — `exam_history_items` (수험생)
```json
{
  "exam_history_items": [
    {
      "id": "EH_049",                    // EH_### 규칙
      "era": "ancient",                  // ancient|three_kingdoms|goryeo|joseon|modern|contemporary 등
      "era_label": "선사·고조선",
      "title": "고조선의 8조법",
      "summary": "3~5문장 핵심 설명.",
      "key_point": "시험에 나오는 핵심 한 줄.",
      "exam_tip": "암기 포인트 / 출제 경향 한 줄."
    }
  ]
}
```

## 4. 복붙용 생성 프롬프트 (다른 대화에 그대로 붙여넣기)

> 아래에서 필요한 블록 하나를 골라 복사 → 새 대화에 붙여넣고 주제만 바꾼다.
> 생성된 JSON을 해당 폴더에 새 파일(또는 기존 파일 배열에 append)로 저장한 뒤 서버를 재시작하면 자동 적재된다.

### (A) 직장인 영어 테마팩
```
SJ 아카이브 "지식 배달" 영어 테마팩 콘텐츠를 생성한다.
출력은 반드시 아래 스키마의 JSON 한 덩어리만. 설명 금지.

- 위치: public/data/knowledge_db/ 폴더의 .json 파일, 최상위 키는 "english_theme_packs"
- 규칙: id는 전역 고유. 이번 배치는 PACK_EN_<직전 최대번호+1>부터. 표현 id는 "<팩id>_01"…"_05".
       기존에 쓴 id와 절대 겹치지 않게 할 것.
- 한 팩 = 직장인이 실제 쓰는 상황 테마 1개 + 표현 5개 + 마스터 단락.
- 주제: <여기에 주제. 예: "재택근무·하이브리드 근무 커뮤니케이션">
- 표현 5개는 서로 다른 뉘앙스, dialogue는 3~4턴 자연스러운 회사 대화(\n 줄바꿈).
- nuance는 "왜 이 표현인가"를 2~4문장 스토리텔링. master_paragraph는 5개 표현을 엮은 5~7문장 + 한국어 번역 + highlights 배열.

스키마:
{ "meta": {"topic":"...","created":"YYYY-MM-DD"},
  "english_theme_packs": [ { "id":"PACK_EN_###","theme_key":"...","theme_title":"...","theme_title_en":"...",
    "level":"intermediate","delivery_date":null,
    "expressions":[ {"id":"PACK_EN_###_01","order":1,"expression":"...","meaning":"...","nuance":"...","dialogue":"...","example":"...","practice":"..."} ],
    "master_paragraph":{"text":"...","translation":"...","highlights":["..."]} } ] }
```

### (B) 수험생 수능 영단어 팩
```
SJ 아카이브 수험생 "수능 영단어" 콘텐츠를 생성한다. JSON 한 덩어리만 출력, 설명 금지.

- 위치: public/data/exam_db/ 폴더의 .json, 최상위 키는 "exam_vocab_themes"
- 규칙: 팩 id는 EV_PACK_<직전 최대+1>, 단어 id는 EV_<직전 최대+1>부터 전역 고유. 기존 id와 겹치지 말 것.
- 한 팩 = 출제 유형/주제 1개 + 단어 5개. csat_ref는 실제 수능 출제 유형/연도.
- 주제: <여기에. 예: "어법성 판단 빈출 동사">

스키마:
{ "meta":{"topic":"...","created":"YYYY-MM-DD"},
  "exam_vocab_themes":[ {"id":"EV_PACK_###","theme_title":"...","level":"high","tip":"...","delivery_date":null,
    "words":[ {"id":"EV_###","order":1,"word":"...","pos":"...","meaning":"...","example_en":"...","example_ko":"...","csat_ref":"...","synonyms":"..."} ] } ] }
```

### (C) 수험생 한국사
```
SJ 아카이브 수험생 "한국사" 콘텐츠를 생성한다. JSON 한 덩어리만 출력, 설명 금지.

- 위치: public/data/exam_db/ 폴더의 .json, 최상위 키는 "exam_history_items"
- 규칙: id는 EH_<직전 최대+1>부터 전역 고유. 기존 id와 겹치지 말 것.
- 시대순으로, 수능 한국사 빈출 주제 위주. summary 3~5문장 + key_point/exam_tip 각 한 줄.
- 범위: <여기에. 예: "고려 후기 사회·경제">

스키마:
{ "meta":{"topic":"...","created":"YYYY-MM-DD"},
  "exam_history_items":[ {"id":"EH_###","era":"goryeo","era_label":"고려","title":"...","summary":"...","key_point":"...","exam_tip":"..."} ] }
```

## 5. 추가 후 체크리스트
```
□ 올바른 폴더에 넣었는가? (직장인=knowledge_db, 수험생=exam_db)
□ 최상위 키 이름이 정확한가? (오타 시 미적재)
□ id가 기존과 안 겹치는가? (겹치면 INSERT OR IGNORE로 누락)
□ JSON 문법 OK? (node -e "require('./파일.json')"로 확인)
□ 서버 재시작 후 콘솔에 "[EnTheme Seed] N개" / "[ExamKnowledge Seed] ..." 로그로 적재 확인
```

> 현재 직전 최대 id 확인 방법:
> `knowledge_db`/`exam_db`의 마지막 배치 파일에서 가장 큰 번호를 보거나, 서버 콘솔의 시드 로그 개수로 가늠.
